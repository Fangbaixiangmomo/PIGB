# Physics-Informed Finite-Basis Experiments

This repository contains the code used to reproduce the numerical experiments
for the thesis paper:

> Physics-Informed Finite-Basis Estimation and Inference for Noisy
> Option-Pricing Surfaces

The experiments study finite-basis price-surface estimation for option-pricing
PDEs.  The main scripts compare raw basis regression, payoff-informed baselines,
PDE-regularized finite-basis estimators, predictive benchmarks, coverage
diagnostics, Boulevard convergence, and a real SPY option-chain application.

## Repository Layout

```text
experiments/                 Production scripts used for the paper
src/                         Shared pricing, basis, estimator, and metric code
results/raw/                 Generated raw simulation outputs and cleaned data
results/tables/              Generated CSV tables for the paper
results/figures/             Generated PNG/PDF figures for the paper
```

The `results/` directory is generated by the scripts and is ignored by git.
Local scratch notebooks, exploratory scripts, and generated outputs are also
ignored; they are not needed for paper replication.

## Environment Setup

Use Python 3.10 or newer.  The current experiments were run with Python 3.12.

```bash
python -m venv pigb_env
source pigb_env/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

The main dependencies are NumPy, SciPy, pandas, matplotlib, scikit-learn,
XGBoost, CatBoost, and PyTorch.  Neural-network benchmarks can be skipped with
script flags if PyTorch runtime is a bottleneck.

## Full Paper Replication

Run all commands from the repository root.

```bash
python experiments/main_finite_basis.py
python experiments/coverage_subsection.py
python experiments/boulevard_convergence.py
python experiments/benchmark_horserace.py
python experiments/real_data_options.py
```

This regenerates the main tables and figures under:

```text
results/tables/
results/figures/
results/raw/
```

The scripts set fixed random seeds by default, so repeated runs should be
stable up to ordinary numerical and library-version differences.

### Coverage and Approximation Bias

```bash
python experiments/coverage_subsection.py
```

This script reports pointwise price interval coverage.  Coverage is computed
for both the finite-basis oracle target and the true Black-Scholes surface; it
is not simultaneous surface coverage and not direct coefficient-vector

### Boulevard Convergence

```bash
python experiments/boulevard_convergence.py
```

This verifies the finite-dimensional Boulevard characterization
\(\theta_b \to \{\nu/(\nu+1)\}\theta_{\mathrm{ERM}}\), and shows that the
rescaled coefficient converges to the once-for-all finite-basis ERM solution.
It generates:

```text
results/tables/boulevard_convergence.csv
results/tables/boulevard_convergence_summary.csv
results/figures/boulevard_convergence_rescaled.png
results/figures/boulevard_convergence_rescaled.pdf
results/figures/boulevard_convergence_diagnostics.png
results/figures/boulevard_convergence_diagnostics.pdf
```

Useful options:

```bash
python experiments/boulevard_convergence.py --iterations 300
python experiments/boulevard_convergence.py --nu 0.5 --n-train 1000
```

### Predictive Horse-Race Benchmarks

```bash
python experiments/benchmark_horserace.py
```

This compares finite-basis estimators with a finite-difference PDE oracle,
random forests, XGBoost, CatBoost, a pure neural network, and a PINN on
simulated call and digital-call data.  The PDE solver is an oracle reference
because it uses the true Black-Scholes parameters; it is not a noisy-data
competitor.  It generates:

```text

results/tables/benchmark_horserace_robustness_raw.csv

results/figures/benchmark_horserace_accuracy_runtime.pdf
results/figures/benchmark_horserace_robustness.png
```

### Real SPY Option-Chain Experiment

```bash
python experiments/real_data_options.py
```

This script uses a local Cboe SPY option-chain snapshot and fits calls and puts
separately across maturity buckets.  The empirical section focuses on vanilla
calls and puts because liquid public option-chain data do not provide quoted
digital or other exotic contracts in the same clean format.  The script reports
held-out price accuracy and implied-volatility diagnostics.

Expected input:

```text
results/raw/cboe_options_SPY_raw.csv
```

Generated outputs:

```text

results/raw/real_data_iv_surface_grid.csv
results/tables/real_data_sample_summary.csv

results/figures/real_data_benchmark_by_slice.png
results/figures/real_data_benchmark_by_slice.pdf
results/figures/real_data_iv_smiles.png
results/figures/real_data_iv_smiles.pdf
results/figures/real_data_iv_surface.png
results/figures/real_data_iv_surface.pdf
```

Optional PINN run:

```bash
python experiments/real_data_options.py --include-pinn
```

If the SPY raw file is missing, download or otherwise place a Cboe-format
snapshot at `results/raw/cboe_options_SPY_raw.csv`.  The older helper script

schema, but the paper replication uses the saved local snapshot so that the
empirical sample does not change over time.

## Notes on Reproducibility

- Production scripts live in `experiments/`.
- Output paths are relative to the repository root.
- Most scripts create `results/raw`, `results/tables`, and `results/figures`
  automatically.
- Full robustness and neural-network benchmarks are the slowest parts of the
  replication.  Use the quick flags above while developing, but use the default
  commands for paper tables and figures.
- Real-data results depend on the fixed local SPY snapshot.  Downloading a fresh
  Cboe file will change the empirical sample and therefore the reported numbers.
